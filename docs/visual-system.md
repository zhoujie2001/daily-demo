# Sunset Paper Visual System

P1 将全站统一为“日落纸张”视觉语言：温暖、克制、带有编辑排版感，同时让功能控件保持清晰。

## 设计原则

1. 内容优先：大标题建立章节节奏，正文与媒体保持足够呼吸空间。
2. 暖而不黄：纸张色只承担氛围，正文始终使用高对比炭黑。
3. 陶土色只做提示：用于编号、当前状态、焦点和关键反馈，不大面积填充。
4. 表面层级不超过三层：纸张背景、普通卡片、弹窗或悬浮层。
5. 圆角保持四级：`6 / 10 / 14 / 20px`，胶囊控件使用 `999px`。

## 核心令牌

令牌位于 `src/design-system.css` 的 `:root`。

| 类型 | 令牌 | 用途 |
| --- | --- | --- |
| 背景 | `--paper` | 页面纸张背景 |
| 表面 | `--surface` | 普通卡片、内容容器 |
| 高层表面 | `--surface-elevated` | 弹窗、浮层、输入框 |
| 主文字 | `--ink` | 标题与重要正文 |
| 次文字 | `--ink-muted` | 描述、日期、元信息 |
| 品牌色 | `--accent` | 编号、焦点、当前状态 |
| 分隔线 | `--line` | 边框与章节分隔 |
| 阴影 | `--shadow-sm/md/lg` | 普通、悬浮、弹窗三层高度 |

## 排版

- 展示标题：Cormorant Garamond。
- 正文与控件：DM Sans，并保留系统字体回退。
- 数据、日期和编号：等宽字体。
- 每个内容章节使用 `SectionHeading`，包含两位编号、英文标题和一句中文说明。

## 组件约束

- 新按钮优先复用 `Button`，或遵循 `.ui-btn` 的高度、圆角和反馈。
- 新表单控件使用 `.ui-input`，不要新增硬编码阴影与焦点颜色。
- 新卡片使用 `--surface + --line + --radius-md/lg + --shadow-sm`。
- Hover 位移最多 `2–5px`；移动端不依赖 hover 承载必要操作。
- 新颜色必须先判断能否由现有语义令牌表达，避免直接写十六进制颜色。

## 响应式规则

- `>= 1000px`：固定侧栏和内容区双栏骨架。
- `< 1000px`：固定顶部栏与抽屉导航。
- `< 700px`：单列卡片，章节标题和操作按钮压缩。
- 所有页面必须在 `390 × 844` 下无横向页面溢出。

## 可访问性与动效

- 交互元素统一使用陶土色焦点环。
- 正文与背景保持高对比度，次要文字不用于关键操作。
- `prefers-reduced-motion: reduce` 下关闭非必要动画和过渡。
